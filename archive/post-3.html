How languages do functions.

Preface

Hello everyone, today's post is another ramble of mine about language design.

It's like the title says, how languages define functions and why I find them interesting.

I mainly find them interesting because of the many paradigms that programming languages have, a lot of interesting ideas get thrown around and I'd like to explore some of them.

Examples

Let's start off with good old C: 

C void foo() {
    printf("bar");
}

For most programmers, nothing too unusual. The type is in front of the function name, then some parenthesis if you want to add optional parameters, then an opening curly brace then the function body then the closing brace.

Most C-like languages operate like this, so I won't be discussing them in detail. but I will give some examples. 

JavaScript function foo() {
    console.log("bar");
}

In JavaScript you see that the type is inferred and instead we use the function keyword. 

Python def foo():
    print("bar")

In Python the type is also inferred but since python uses indentation instead of braces, the function is defined with the start of a :.

NOTE: Some programmers argue that Python is not "C-like", but for the sake of this article, let's just say it is 
Zig fn foo() void {
    std.debug.warn("bar");
}

Zig's way of defining syntax is strikingly similar to how C does it, with only a few differences, with the type being behind the opening brace, and an additional fn keyword.

Let's move to something less like C. 

Clojure (defn foo
    [& args]
    (println "bar"))

Ah, Clojure, alright.

In practically all Lisps, everything is contained in parentheses, In Clojure, we first do an opening parenthesis, then the defn keyword, after that the name of the function, then the function arguments, then we get to the function block, which is straightforward, it prints "bar", and it starts and ends with another pair of parentheses, then the closing parenthesis, ending the function.

Let's move on to the final 2 Languages: Haskell and Oberon.

Let's start with Haskell: 

Haskell foo = do
    putStrLn "bar"

As a functional language, Functions are Haskell's main focus (obviously), nothing too remarkable, I would explain it, but I do feel it's self-explanatory, if you want a good code example of Haskell, I recommend this article.

Time for Oberon. 

Oberon PROCEDURE foo*;
BEGIN
    Out.Open;
    Out.String("bar");
END

In Oberon, functions are called procedures, which in all honesty is a better name. Some programmers may notice that the language looks a bit like PASCAL, that is because the language was created by the same person, Niklaus Wirth. We first start the procedure by typing PROCEDURE then the procedure name then a * then a ;, then we BEGIN, open the output, then we output the string "bar", after all of that, we end the procedure.

Conclusion

I hope this article helped you understand why I find this all interesting, different spoken languages do different things, same with programming languages, they may have similarities, but most of them have differences in some way, and that concludes the article, I hope to see you next time.
