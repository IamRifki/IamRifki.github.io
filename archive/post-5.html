Haskell: Learning and Loving.

Preface

Haskell is a functional programming language, it means it's not an imperative programming language like C for example.

Haskell is very interesting to me because instead of telling the computer what it should do, instead you tell it abstract things about the things it should do, I'll explain in the next section.

Main :: IO ()

Let's talk about how an imperative programmer would implement walking.

Imperative programmers would say that walking is moving, but Haskell programmers would say that Walking is moving each feet in front of the other feet.

It's a bit nonsense at first, but as you program in Haskell more, it seems natural.

Haskell (when used correctly) also reduces the amount of code written, let's take a look at some Examples.

Examples

C++ #include <iostream>
#include <cmath>
#include <cstdlib>

using namespace std;

bool isPrime(int);

int main(int argc, char* argv[]) {
    int primes = 10000, count = 0;
    if (argc > 1) {
    primes = atoi(argv[1]);
    }
    if (isPrime(2)) {
    count++;
    }
    for (int i = 3; i <= primes; i+=2) {
        if (isPrime(i)) {
            count++;
        }
    }
    cout << count << endl;
    return 0;

}

bool isPrime(int x){
    for (int i = 2; i <= floor(sqrt(x)); i++) {
        if (x % i == 0) {
            return false;
        }
    }
    return true;
}

Now, that doesn't look long for the average imperative programmer, but let's take a look at the Haskell example. 

Haskell prime :: Int -> Bool
prime x = go 3
    where
        go q | q <= isqrt x = if rem x q == 0 then False else go (q+2)
        go _ = True

That's way nicer and cleaner.

Overall the code in Haskell is much cleaner, readable, and takes less time to code.
