Deno and the Future of JavaScript runtimes.

Prelude

When I was starting JavaScript in Node.js 3 years ago, I never really liked it, the package management is awful, with this weird package.lock system, and it unfortunately carried over to languages like Rust.

When I first heard about Deno, I wasn't all that thrilled, at first it just looked like a rewrite of Node.js in Rust, but I checked it even more and now I see it as a replacement for Node and a saviour to people who are frustrated with Node.

A bit of history

Deno and Node.js were made by the same person, Ryan Dahl. He originally created Node to make so that the backend and frontend of an application can be written in one language, which in itself is a good thing.

However Dahl made a lot of wrong decisions and Node has become infamous for callback hell, and the messy NPM package management which encourages a billion small packages like is-number to pop up, and this caused a lot of problem when the left-pad package came around and was removed from the NPM registry, causing the first signs of the collapse of civilization.

Eventually in Dahl's 2018 talk 10 Things I Regret About Node.js he rambled about everything he thinks is wrong with Node, and the fact it was too late to fix them meant that he decided to start a new project called Deno (Which is an anagram and alphabetized version of the word Node).

Things I'm glad are fixed

First of all, there's no such thing as a package manager, instead you link DIRECTLY to the .js file that you want to use, Pretty neat eh?, it's just like developing in the browser. plus this discourages the appearance of another left-pad fiasco happening, and this renders using a super small library hard to do instead of just copy-pasting to a utils file.

Second, first class support for Modern JavaScript. ESM Modules, TypeScript, Promises and Async/Await, etc

Third, since first-class JavaScript support is in Deno, there's no need for complex tooling that has a billion abstractions on it, just do deno run. Which means no Babel or any of that nonsense.

Fourth, you might be thinking, all of this?, you might need more than 1 binary, Well guess what?, There's only ONE binary, everything is contained in 1 binary, including the TypeScript compiler!

Fifth, None of those legacy Node.js crap that's been there for years, all of Deno is clean fresh new code that's confined to JavaScript's Standard, plus a couple of features.

Conclusion

I really hope Deno takes off as THE JavaScript runtime because it is just a huge improvement over Node in so many ways.
