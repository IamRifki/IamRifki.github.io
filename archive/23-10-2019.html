A quick overview of Odin's syntax.

Preface

Odin is a programming language created by gingerBill.

It's a very good language and I recommend checking it out.

One of the highlights of this language in my opinion is its Syntax, which looks a bit weird at first, but once you actually write in it, it feels natural so to speak.

Examples

Take this for example: 

Odinimport "core:fmt"

foo :: proc() {
    fmt.println("bar");
}

Notice two things about it, the :: operator and the name proc.

Unlike in C++, the :: operator does not mean scope resolution, instead, it is used to denote the type of in this case a proc.

in most languages procedures are called functions, and they are usually defined like: 

C void foo() {
    printf("bar");
}

or:

Clojure(defn -foo
    [& args]
    (println "bar"))

Calling what other languages call functions "procedures" may look like it's just trying to break the C tradition, but there is a real explanation said by gingerBill himself.

in Odin the :: operator not only denotes procedures but also constants and structs. 

Odin/* constants */
foo :: 123;
bar : int : 123;

/* variables */
foo := 123;
bar : int = 123;

In the constants examples, foo's value is untyped and bar's value is typed. You do not need to type them, they also serve as the equivalent of a C macro, so instead of this in C: 

C#define CONSTANT 123

In Odin, you would write: 

OdinCONSTANT :: 123;

In the variable examples, foo and bar both have the same type despite being written differently, this is because of Odin has type inference similar to Go.

Structs example: 

Odinfoo :: struct {
    bar: f32,
    baz: f32,
}

and the list continues.

Opinions / My experience with Odin

At first, I do find this a bit jarring, coming from languages like C, Go, Python, Lisp, etc.

but, as I used the language more, the syntax feels natural, as I said earlier.

Conclusion

Language design is an interesting topic that I would like to explore again some time, I have written a programming language before, but I really didn't like how the syntax turned out. Odin proves that a single man with some help can design a language and exceed other languages by at least a mile.

I hope this article can raise awareness to Odin and why it's interesting, until next time, see you all around.
